create a servo served by background processing

Construct a servo from the output pin, the servo It writes a pulse to servo each 20 ms.

During the pulse, it can either

  • use busy busy waiting, which will block other background processing for the duration of the pulse;
  • return and wait for the next activation, which will cause jitter when the application spends significant time without calling a (non-busy) wait function. The default is to use busy waiting.
