Detailed Description

background processing

This class implements run-to-completion style background processing.

A class that needs background processing must inherit from background and implement the work function. This work function will be called from the wait_s(), wait_ms(), wait_us() etc. functions.

When an application contains background work, all plain wait functions can take longer than the specified time, up to the run time of the longest runtime of the work() functions.

No background work will be done from delay calls made while a work() function is running.

For all background jobs: be careful to preserve the object!
